01Architecture refactor — Grid Terminal across all of /v2

Lifted the floating nav and footer out of the homepage into a shared shell, then mirrored that chrome on every inner page so the site reads as one coherent product.

  • Extracted V2Shell (v2-shell.tsx) — owns the floating nav, page-wide pixel pattern, scan line, and footer. Mounted from /v2/layout.tsx so every /v2/* route inherits identical chrome.
  • Stripped inline nav/footer from /v2/page.tsx; the homepage body is now content-only.
  • Built four new inner pages in Grid Terminal style: /v2/services, /v2/about, /v2/case-studies, /v2/security (later joined by /v2/blog, /v2/faqs, /v2/contact).
  • V2InnerHero + V2InnerCta shared primitives so every inner page's hero block and bottom CTA stay visually consistent.
  • V2Link routing helper — uses next/link for /v2/* paths (which sit outside the i18n routing tree) and next-intl's locale-aware Link for everything else. Caught a class of bugs where Grid Terminal links were 404ing as /en/v2/services.
  • Wrapped the public homepage with V2Shell so /, /en, /ar get the chrome the same way /v2/* does — fixed a regression where the homepage navbar looked "gone" after the refactor.
  • Removed the legacy /v2/(site)/* re-export tree.

09Bug fixes worth remembering

Production env var had a trailing newline prod-only

NEXT_PUBLIC_API_URL was set as https://api-steel-five-95.vercel.app/v1\n, so every client-side fetch hit a malformed URL and silently failed → fallback content rendered. All admin services / FAQs / stats / blog were silently broken on prod for a while. Fix: removed via vercel env rm and re-added with printf (no trailing newline) instead of echo.

/en/v2/... 404s

Multiple links built via next-intl's locale-aware Link were producing /en/v2/contact etc., which 404'd because /v2/* sits outside the i18n routing tree. Fix: created V2Link helper that picks next/link for /v2/* destinations and next-intl's Link for everything else. Swapped at every callsite that touches a /v2/* href.

"Open engagement" CTA looped back to the same page

The V2InnerCta button on /v2/contact was hardcoded to /v2/contact. Added optional href + label overrides so the contact page passes mailto:<adminEmail>, while every other inner page keeps the default behaviour.
